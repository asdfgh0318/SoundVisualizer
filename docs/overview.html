Locked decisions

No azimuth. Tested objects are rotationally symmetric around the vertical axis. Elevation is the only spatial dimension. The 360° polar view is purely cosmetic mirroring of the 180° measured arc.
6 mics, 2 configurations. "Top half" / "bottom half", both fully user-defined per-mic in Setup. The user manually re-mounts mics between captures. No hardcoded pairing.
Filesystem JSON+WAV store. No SQL. Keys are motor__propeller__shroud__notes directories under data/.
Paweł's code is ours. Full permission granted by the user (2026-05-06) to copy/edit. msp.py, thrust_stand.py, norsonic*.py reused wholesale.
Calibration constants are unit-specific. Paweł's HINGE_DISTANCE, CAL_* values are correct for this user's 1585 but get exposed via config.toml rather than hardcoded.
Trigger-onset sync stays. UMIK-2s cannot be hardware-clock-locked. Existing JS implementation moves into Python server-side — no new alignment strategy.
Cutoff triggers: all 8 channels. Current, voltage, RPM, thrust, torque, temp0/1/2 — each tickbox-enabled with threshold. Server watchdog at ~33 Hz slams PWM to 1000µs on any trip.
FFT defaults. Hann window, 4096 size, 50% overlap, log-x 20 Hz–24 kHz, dBSPL after UMIK calibration (dBFS fallback). 48 kHz sample rate.

# Data structure
Our database should be a structure mapping key structure to a set of measurements. The key structure should be composed of following string fields: motor, propeller, shroud, notes. A measurement should be a structure representing a single measurement, either with accustic data from a single microphone, or with performance data from thrust stand. All measurements should have fields with the exact timestamp of its start and end. The type must be disinguisible by some mechanism (either dedicated type field, or just python object type). performance measurements should have all performance-related data. Accustic measurements should have all appropiate accustic data along with operating conditions info (angle of the microphone it was taken at), might also have some metadata about which microphone it was and/or what calibration data was used. Form of storage of the database is up to your choice, but must be consulted with me first.
